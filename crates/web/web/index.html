starstream-run

Load, link and instantiate Starstream contracts in the browser — each runs in its own Web Worker via wasmtime's Pulley interpreter.

Accepts a full component or a core module with a component-type custom section. Each loaded contract gets its own Web Worker and is identified by its sha256; loading a contract whose hash is already present is rejected. Within a contract's card, every exported instance that owns a resource is listed: call a constructor (e.g. new) or load from storage to mint a live UTXO (labelled <instance> #<n>). Then use the single Invoke panel at the top — pick a contract, pick one of its live UTXOs, and call that UTXO's methods; return values and emitted events stream into that panel's log. A method is only callable once the UTXO has declared it via implements-method during its constructor — methods it has not declared are shown disabled. Per-contract trace logs appear at the bottom of each card.
You can also connect to a UTXO served by a starstream-run CLI (started with --serve <ADDR>): enter its address above and click Connect. The page fetches the served WIT over HTTP, lists the served UTXO and its methods in the same Invoke panel, and invokes them over WebSockets via wRPC — the CLI runs the guest and owns the UTXO; nothing is instantiated in the browser. A served contract has no constructors or storage to mint from: the CLI already minted the single UTXO it serves. When the page is itself served over https, the CLI must be reachable over wss; the plain-text CLI is intended for local (http) use.
